# TasksACE Rules & Documentation

This folder contains important documentation and guidelines for maintaining consistency across the TasksACE website.

## 📁 Contents

### 📄 URL-Slug-Documentation.md
**Complete migration history and current URL structure**
- Before/After comparison of all URL changes
- Technical details of the migration
- Complete list of all current URLs
- SEO improvements achieved

### 📄 URL-Generation-Rules.md  
**Guidelines for creating new URLs**
- Step-by-step process for new services
- Naming conventions and best practices
- Common scenarios and solutions
- Implementation checklist

## 🎯 Purpose

These documents serve as:
- **Historical Record** - Track all URL changes made
- **Reference Guide** - Quick lookup for current URLs
- **Standards Manual** - Ensure consistency for new services
- **Training Material** - Help developers understand URL strategy

## 🔄 When to Update

### URL-Slug-Documentation.md
Update when:
- Any URL structure changes are made
- New services are added
- Migration projects are completed

### URL-Generation-Rules.md  
Update when:
- New URL patterns are established
- Best practices evolve
- New service categories are created

## 👥 Who Should Use This

- **Developers** - When adding new services or pages
- **Content Managers** - When organizing site structure  
- **SEO Team** - For understanding URL strategy
- **Project Managers** - For planning new features

## 🚀 Quick Start

**For adding a new service:**
1. Read URL-Generation-Rules.md
2. Follow the decision tree
3. Check against existing URLs in URL-Slug-Documentation.md
4. Implement following the checklist

**For understanding current structure:**
1. Check URL-Slug-Documentation.md
2. Find your service category
3. See current URL patterns

## 📞 Questions?

If you're unsure about URL decisions:
1. Check the documentation first
2. Follow the established patterns
3. When in doubt, choose the simpler option
4. Document any new patterns you create
